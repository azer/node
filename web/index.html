NodeJs Library In Client-Side

This page demonstrates client-side usage of some of the modules in NodeJS library.

Why?

NodeJS' library contains some modules (e.g assert) that we need to use in client-side, also. I especially want to use crypto module in client-side and will give a try to emscripten this week.

  • Browse the Source Code

How It Works?

The only difference between the backend platforms CommonJS designed for and client-side environments is that web browsers don't provide no module/package mechanism.

In brief, browsers are able to take the goodness of CommonJS that defines a well designed module and package mechanism for JavaScript. Only thing we need to do is to change the module patterns we used to code.

To make this example, I used JSBuild to generate the wrapper code and combine the files.

Check out the below example using Firebug Lite window.

Available Modules:

  • assert
  • freelist
  • path
  • string_decoder
  • url
  • util

Try It

Open your console (see the Firebug Lite logo on right bottom) and call the available modules listed above. Here is an example usage; 
        > var assert = node.require('assert');
        > assert.equals('foo', [3,1,4]);
        AssertionError: AssertionError: [3,1,4] == 'foo'
